CSS的单位

发表于2023-01-02|更新于2023-01-02|学习笔记
|字数总计:2.8k|阅读时长:9分钟

前言

说起CSS的长度单位，我们在平时开发中可能使用更多的是像素单位（px），它是绝对单位。当我们为一个属性设置了像素值，这个值是是不会修改的。CSS的长度单位可以分为绝对单位、相对单位。

绝对单位

先来看一下绝对单位，绝对单位的长度是固定的，用任何一个绝对单位表示的长度都将恰好显示为这个尺寸。绝对单位中使用最多的就是px了。还有几个不常用的决定单位，例如：mm（全称为 Millimeters，表示毫米）、cm（全称为 Centimeters，表示厘米）、in（全称为 Inches，表示英寸）、pt（全称为 Point，表示点，印刷术语，1/72英寸）、pc（全称为 Picas，表示派卡，印刷术语，12点）

以上几个单位的换算如下：1in = 25.4 mm = 2.54 cm = 6pc = 72 pt = 96px。

相对单位

再来看一下相对单位，相对单位的值会根据外部因素发生变换。相对单位大致可以分为两类，一类为字体相对单位，另一类为视窗相对单位。

  • 字体相对单位，它们都是根据font-size来进行计算的。常见的字体相对单位有：em、rem、ex、ch
  • 视窗相对单位，它们都是根据视窗大小来决定的。常见的视窗相对单位有：vw、vh、vmax、vmin。

下面来看一下这些单位

em和rem

em

em是最常见的相对长度单位，适合基于特定字号进行排版。在CSS中，1em等于当前元素的字号，其准确值取决于作用的元素。

在上图的例子中，在父元素的 div 中设置字体大小为 16px ，也就是 1em 的值。然后设置 .wrap 的 padding 值为 1em 。浏览器在渲染时，将 em 的数值乘以字体的值，最终得到一个绝对值16。

那假如我们设置 .inner 的内边距为 2em ，那么它最终计算出来的绝对值为多少呢？

答案是 32px 。这里是因为子节点继承父节点的 font-size 值，1em = 16px ，所以 2em 就等于 32px 。

那假如我们修改了父元素 .wrap 的 font—size 值为 20px，该节点及其子节点的 padding 均会发生变化。

如果我们再在 .wrap 节点上添加一个父节点，同样为父节点设置 padding : 1em，此时父节点计算出来的相对值是 16px 。为什么不是20px呢？原因很简单，我们没有为 .wrap 节点的父节点设置 font-size 值，它会继承 body 的 font-size 值，而 body 的默认font-size = 16px，所以对于它的子节点来说，1em 就等于16px。

总结一下，如果自身元素是没有设置字体大小的，那么就会根据其父元素的字体大小作为参照去计算，如果元素本身已经设置了字体，那么就会基于自身的字体大小进行计算。

rem

讲完 em 后，再来看看和它类似的 rem ，em 是根据当前元素的 font-size 进行计算，而 rem 则是根据根元素的 font-size 进行计算。在浏览器解析HTML文档时，会在内存里将页面的所有元素表示为DOM（文档对象模型）。它是一个树状结构，其中的每一个元素都是由一个节点表示，刚才提到的根元素就是是指

元素。

将em所使用案例的 em 全部替换成 rem ，我们会发现 1rem=16px 。如果想修改 rem 的值，只能修改根元素的 font-size 值才有效。根节点有一个伪类选择器（:root），可以用来选择它自己。这等价于类型选择器 html 。

小结

使用 em 和 rem 可以让我们灵活的够控制元素整体的放大和缩小，而不是固定大小。那何时应使用 em，何时应使用 rem 呢？可以根据两者的差异来进行选择：

  • 两者在页面中计算出来的样式都会以px的形式显示
  • rem是相对于根元素html的font-size计算，em 相对于元素的font-size计算
  • 当需要根据浏览器的font-size设置缩放时，应该使用 rem
  • 使用 em 应该根据组件的font-size来定，而不是根元素的font-size来定
  • rem 可以从浏览器字体设置中继承 font-size 值， em 可能受任何继承过来的父元素 font-size 的影响

ex和ch

ex 和 ch 都是排版用的单位，它们的大小取决于元素的font-size 和 font-family属性。

  • ex 指的是所用字体中小写字母 x 的高度。因此，如果两个字体不一样，那么 ex 的值是不一样的。因为每种字体的小写 x 的高度是不一样的。
  • ch 和 ex 类似，不过它是基于数字 0 的宽度计算的。会随着字体的变化而变化。而0 的宽度通常是对字体的平均字符宽度，它是一个估计值。由于 ch 是一个近似等宽的单元，所以在设置容器的宽度时很有用，比如一个容器想要显示指定个数的字符串时，就可以使用这个单位。

vw、vh、vmax 和 vmin

这四个单位都是视口的相对单位，所谓视口，就是浏览器窗口里网页可见部分的边框区域。视口不包括浏览器的地址栏、工具栏、状态栏。如果视口大小发生了变化，那么这些值都会随之变化。

单位作用
vw相对于视口宽度的 1%
vh相对于视口高度的 1%
vmax相对于视口较大尺寸的 1％
vmin相对于视口较小尺寸的 1％

举个例子，50vw 等于视口宽度的一半，25vh 等于视口高度的四分之一。vh 和 vw 的值总是和视口的高度和宽度有关。

vmin 取决于视口宽和高较小的一方，vmax则取决于视口宽和高较大的一方。假如一个页面的宽高分别为1200px和600px，1vmin就等于6px，1vmax就等于12px。

百分比

百分比（%）也是我们比较常用的单位之一，所有接受长度值的属性都可以使用百分比单位。但是不同属性使用该单位的效果可能并不一样。

盒模型中的百分比

在CSS中的盒模型包含的属性有：width、max-width、min-width、height、max-height、min-height、padding、margin等。这些属性在使用百分比时，参照物不尽相同：

  • width、max-width、min-width：相对于包含块的 width 进行计算

  • height、max-height、min-height：相对于包含块的 height 进行计算

  • padding、margin：如果是水平的值，就是相对于包含块的 width 进行计算；如果是垂直的值，就是相对于包含块的 height 进行计算。

定位中的百分比

在CSS中用控制 position 位置的top、right、bottom、left都可以使用百分比作为单位。其参照物就是包含块的同方向的width和height。不同定位的包含块不尽相同：

  • 如果元素为静态（ static ）或相对定位（ relative ），包含块一般是其父容器

  • 如果元素为绝对定位（ absolute ），包含块应该是离它最近的 position 为 absolute 、 relative 或 fixed 的祖先元素

  • 如果元素为固定定位（ fixed ），包含块就是视窗（ viewport ）

字体中的百分比

在CSS中文本控制的属性有font-size、line-height 、vertical-align、 text-indent等。这些属性在使用百分比时，参照物不尽相同：

  • font-size：根据父元素的font-size 进行计算
  • line-height ：根据font-size进行计算
  • vertical-align（vertical-align 属性设置元素的垂直对齐方式）：根据line-height进行计算
  • text-indent（text-indent 属性规定文本块中首行文本的缩进）：如果是水平的，则根据width进行计算，如果是垂直的，则根据 height 进行计算

变换中的百分比

CSS 中的 transform 属性中的 translate 和 transform-origin 值也可以设置百分比。

  • translateX() 根据容器的 width 计算
  • translateY() 根据容器的 height 计算
  • transform-origin 中横坐标（ x ）相对于容器的 width 计算；纵坐标（ y ）相对于容器的 height 计算

在 translate 还有一个 z 轴的函数 translateZ() 。它是不接受百分比为单位的值

无单位的数值

第一种情况是，如果长度为0，则不需要单位标识符。否则，单位标识符是必需的，不区分大小写，并且必须紧跟在值的数字部分之后，中间没有空格。因为是0，所以0px = 0% = 0em

一个无单位的0只能用于长度值和百分比，比如内边距，边框，宽度，高度等，而不能用于角度值、时间值等。如果没有正确使用0则会导致表达式失效

当然还有一种就是表示行高时，可以使用无单位的数值。700 等于 blod，400 等于 normal。

拓展

CSS 的单位除了绝对长度单位和相对长度单位还有时间单位，角度单位，分辨率单位。

时间单位

CSS中的时间单位有两个：秒（s）和毫秒（ms）。这两个时间单位都是CSS3新增的单位。这两个单位的换算关系如下：1s = 1000ms

时间单位主要用于过渡动画中。

角度单位

CSS中的角度单位有四个：deg、grad、rad、turn。这些角度单位都是CSS3中新增的单位。角度单位常用于旋转，颜色渐变中。

deg 全称为 Degress，表示度，一个圆总共360度。

grad 全称为 Gradians，表示梯度，一个圆总共400梯度。

rad 全称为 Radians，表示弧度，一个圆总共2π弧度。

turn 全称为 Turns，表示圈（转），一个圆总共一圈（转）。

于是就有了以下的换算关系：1turn = 6.2832rad = 400grad = 360deg

分辨率单位

CSS中的分辨率单位有三个：dpi、dpcm、dppx。这三个单位都是CSS3中新增的单位。他们都是正值，不允许为负值。这三个单位的换算关系如下：1dppx = 96dpi 1dpi ≈ 0.39dpcm 1dpcm ≈ 2.54dpi

分辨率单位主要用于媒体查询中。

dpi 全称为 dots per inch，表示每英寸包含的点的数量。普通屏幕通常包含 72或96个点，大于 192dpi 的屏幕被称为高分屏。

dpcm 全称为 dots per centimeter，表示每厘米包含的点的数量。

dppx 全称为 dots per pixel，表示每像素（px）包含点的数量。由于CSS px的固定比率为1:96，因此1dppx相当于96dpi。它对应于由图像分辨率定义的CSS中显示的图像的默认分辨率。
